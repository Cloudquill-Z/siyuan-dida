# 思源滴答同步

这是一个仅面向思源桌面端的插件，用于通过本机 `dida` CLI 同步 Markdown 待办块和滴答清单。

当前处于 MVP 开发阶段。

## 同步策略

- 每个同步范围按批处理思源中的候选待办块，默认每批 200 个。
- 插件会为每个同步范围记录游标：本轮处理一批后，下轮从后续候选块继续；扫到末尾后再回到开头。
- 已确认完成同步的任务会标记为 `completed-synced`，之后默认不会再进入已完成候选批次，避免长期占用每批名额。
- 如果已归档的完成任务在思源中重新改成未完成，它会重新成为普通候选任务并继续同步。
- 已绑定的未完成待办如果标题在思源中变更，会同步更新滴答清单中的任务标题。

## Windows 使用提示

- 先在 Windows 终端确认 `dida --version` 可用；如果思源找不到命令，在插件设置里填写 `dida.cmd` 的绝对路径。
- `dida auth login` 保存的 token 需要在每台电脑各登录一次。
